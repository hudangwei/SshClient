VNC CLIENT Screen

This screen is used to connect to a VNC server located on a host accessible via SSH. The VNC connection will be transported via a temporary SSH tunnel.

Notes:

  • You do not need to create a tunnel via the TUNNELS screen. The VNC screen creates a temporary tunnel itself.
  • Since the connection to the VNC server is via an SSH tunnel, the VNC server port can be firewalled on the server to allow connections only from the local host.

To connect:

  • Switch to the VNC CLIENT screen via the Android menu button -> More -> VNC client.
  • If not already connected, connect to the VNC server system via SSH by putting in the SSH username and hostname in the user@host[:port] box at the top of the screen. You may need to tap in the user@host[:port] box for it to accept entry.
  • Enter the VNC server port number, usually 5900 + screen number, eg, 5907 for screen 7.
  • Enter the VNC password for that screen if required.

To disconnect, click Android menu button -> Discon. To reconnect, click Android menu button -> Connect.

While connected, the screen may be resized or dragged using two fingers. For sending mouse events, use one finger (short tap for left click, long press for right click).

Tapping the screen will also bring the Android keyboard up if it is not present.

The Android's menu contains these main buttons:

  • Ctrl-... - the next character entered on the keyboard will be shifted to control character, if it is an alphabetic. If a non-alphabetic is entered, the Ctrl-... will be abandoned and the character will be sent normally.
  • Ctrl-C - sends a Control-C character to the host.
  • Connect - connect the VNC channel.
  • Discon - disconnect the VNC channel.
  • Paste - sends the contents of a clipboard to the remote host shell.
  • Tab - sends a TAB character to the remote host.
  • More >> - opens up the extended menu.
