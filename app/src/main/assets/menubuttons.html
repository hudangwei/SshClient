Menu Buttons

The Android's menu contains these main buttons:

  • Ctrl-... - the next character entered on the keyboard will be shifted to control character, if it is an alphabetic. If a non-alphabetic is entered, the Ctrl-... will be abandoned and the character will be sent normally.
  • Ctrl-C - sends a Control-C character to the host.
  • Freeze - freezes the text in place on the screen (ie, blocks the host from making updates). While frozen, the foreground/background colors are reversed.
  • Paste - sends the contents of a clipboard to the remote host shell.
  • Tab - sends a TAB character to the remote host.
  • More >> - opens up the extended menu (see below).

... and these extended buttons (accessed by clicking the More button in the main menu):

  • clear screen - clears the SHELL screen text window.
  • clipboard <-> file - transfer clipboard to/from file
  • disconnect - disconnects current session, both shell and file transfer.
  • EXIT - after a typical OK/Cancel dialog, immediately closes connection(s) with remote host(s) if any and closes the app.
  • file transfer - opens the FILE TRANSFER screen if connected or queues it for display if not connected. If this is the first time opening it for this session, the remote home directory is scanned and presented.
  • HELP - displays this HELP screen.
  • known hosts - brings up the Known Hosts menu.
  • local files only - opens a FILE TRANSFER screen for navigating local files without being connected to any remote host.
  • local keypairs - brings up the Local Keypair menu.
  • master password - displays menu for setting/changing master password.
  • saved logins - brings up the Saved Logins menu.
  • sessions - selects different sessions.
  • settings - opens the SETTINGS screen.
  • shell - opens the SHELL screen. If connected and the shell is not open, the shell is opened.
  • tunnels - opens the TUNNELS screen.
